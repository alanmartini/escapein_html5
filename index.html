APIs - Web Sockets

Estabelece conexões entre um navegador da web e um servidor, ambos recebendo e enviando informações. Chat Exemplo

APIs - < video> e < audio>

Uma das maiores vantagens do elemento < video> do HTML5 é que, finalmente, videos passam a ser “integrantes” comum da web. E ainda podemos ‘enfeita-lo’ com CSS (opacidade, posicionamento, transformações) NÃO PRECISA MAIS DE FLASH \o/ PS: É necessário ter um formato de vídeo para cada engine de browser diferente (webkit, trident (IE), Gecko(FF)) O mesmo acontece com o áudio.

APIs - Geolocalização

Através da API é possível obter a localização do usuário independente do dispositivo (desde que o mesmo tenha um browser atualizado) A informação não é retirada de apenas 1 local, um dispositivo móvel pode ser localizado usando o GPS, a localização da antena de celular, informações da antena wi-fi, base dados de endereço IP, o browser irá tratar a informação e repassar as coordenadas.

APIs - Web Storage

Armazenamento de dados associados com uma aplicação é uma atividade fundamental !! 1º Gravação das informações no servidor 2º Gravação as informações em Cookies
  • 3º Web Storage !!
  • Até 5 mb de arquivo
  • Dividido em sessionStorage e localStorage

APIs - Eventos orientados a dispositivos

Muitos dispositivos novos (e alguns antigos) tem hardware desenvolvido para determinar a captura de movimento e informações sobre posicionamento do dispositivo. Esses eventos podem ter muitas utilidades como por exemplo, direcionar objetos no telefone apenas inclinando o dispositivo ou utilizando em conjunto com Geolocalização para um passo a passo mais detalhado.

APIs - Eventos Touch

Com a evolução do desenvolvimento móvel se tornou necessário adicionar uma forma de gerenciar eventos Touch e MultiTouch nos dispositivos. O maior exemplo de utilização de MultiTouch são os jogos dinâmicos, que requerem que o jogador aperte várias áreas ao mesmo tempo.

APIs - Full Screen

Por muito tempo essa opção foi deixada de lado por motivos de segurança, mas com as mudanças nas abordagens das APIs isso foi contornado. Não é necessário deixar toda a sua página em full screen, você pode selecionar o elemento desejado e apenas ele entrará em modo full screen.

Drag N Drop

Já utilizamos a algum tempo em algumas ocasiões, mas com ferramentas de ‘terceiros’ que utilizam jQuery, mas agora com suporte nativo do navegador isso não é mais necessário.
Juntando quase todas essas funcionalidades ( e muito trabalho ) temos como exemplo: Banana Bread

Considerações

HTML 5 ainda não está finalizado, a estimativa do W3C é de meados de 2014, até esta data muitas coisas ainda serão alteradas, corrigidas, adicionadas, padrões totalmente definidos (esperamos que sim pelo menos)

Considerações

O que vai melhorar com a ‘adoção’ completa do HTML5? Diminuir o tamanho dos sites/aplicações, aumentar a performance, diminuir dependências, maior compatibilidade entre dispositivos e códigos com menos /gambiarras/ soluções técnicas de contorno.
